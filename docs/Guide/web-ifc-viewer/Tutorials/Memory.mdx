## How to do it

### Monitoring the memory

ğŸ‘€ [stats.js](https://github.com/mrdoob/stats.js/) is a library to **monitor the memory and performance** of your Three.js app, made by the creator of Three.js himself. Let's go ahead and install with `npm i stats.js`, **import**, **initialize** and **attach it** to our viewer.

```js
import Stats from "stats.js/src/Stats";

const stats = new Stats();
addStats();

function addStats() {
    stats.showPanel(2);
    document.body.append(stats.dom);
    viewer.context.stats = stats;
};
```

ğŸ“‹ During development we don't have to second guess our memory management anymore, because now it's **visualized** for us.

### Dispose memory

âœŒğŸ¥¤ Finally let's create our function to **dispose** memory on command using `dispose()`. Here's how it's done:

```js
function releaseMemory() {
  viewer.dispose();
  viewer = null;
  viewer = new IfcViewerAPI({ container });
  viewer.IFC.setWasmPath("../../../");
  addStats();
}
```

â—â— Keep in mind: Once we call `dispose()`, we need to **re-initialize** our viewer instance (and Stats module)â—â—

âŒ¨ï¸ For this example let's attach our function to a new HTML button.

```html
<input type="button" id="release-button" value="Release Memory" />
```

```js

release_button.addEventListener("click", () => releaseMemory());
```
